OpenAI 更新 Agents SDK：把控制层开源，执行交给合作伙伴

OpenAI 调整了 Agent 的技术架构。简单说：把"控制层"（决定 Agent 下一步做什么）和"执行层"（真正跑代码的地方）拆开。控制层开源；执行层可以委托给 Cloudflare、Modal、e2b、Vercel 等合作伙伴的沙箱（一个隔离的临时运行环境，代码在里面跑，就算出错也不会影响外部系统）。

这件事的意义：过去要做一个"像 Codex 那样会自己写代码、改代码"的 Agent，几乎只能绑在 OpenAI 的基础设施上。现在第三方公司可以用同一套架构做类似产品。竞争焦点因此转移，不再只拼模型能力，而是谁的任务调度更聪明、谁的运行环境更安全、谁的状态管理更稳定。

RIP Pull Requests (2005–2026)：GitHub 允许禁用 PR

GitHub 首次允许用户在开源项目上禁用 Pull Request（PR：开源协作的标准流程，别人把改好的代码以一份 PR 的形式提交，维护者审核后合并）。此前 GitHub 只允许禁用 Issue，不允许禁用 PR。

这件事本身是小新闻，但背后的趋势不小。Peter Steinberger 和 Mitchell Hashimoto 等人已在讨论用"Prompt Request"替代 PR：与其审查 AI 生成的代码，不如审查生成代码的 prompt。没有合并冲突，维护者更容易按自己的方向修改。

GPT-5.4 Pro 解出搁置数十年的 Erdős 数学难题

GPT-5.4 Pro 给出了 Erdős 问题 #1196 的完整证明，用的是 von Mangoldt 函数的一个反直觉解析方法。翻译成外行话：模型选了一条人类数学家通常不会选的路径。

研究者称这可能是第一个被数学界广泛认可的 AI "Book Proof"。"Book Proof"是 Erdős 自己提出的概念，指那种"最优雅、像神写出来的证明"。

重要的不是这道题本身，是模型开始找到人类不会选的路径。

Swyx（Latent Space 联合创始人）：subagent 本质上是优化问题，不是架构问题

Swyx 说他一直在讲"今年是 subagent 之年"（subagent：主 Agent 派出去处理子任务的小 Agent），但他自己纠正了这个说法：subagent 的核心其实是一个优化问题，不是架构问题。怎么让主 Agent 知道什么时候该派出子 Agent、什么时候自己做，本质上是决策树的剪枝（在所有可能的分支里筛掉不划算的那些）。

同一天他还在说 Meta 的 AI 叙事走到了关键节点："看到了 flop（Llama 4），现在正在看 turn"。这是德州扑克的比喻：flop 是前三张公共牌，turn 是第四张。意思是 Meta 的赌注已经下了，Llama 4 是前三张牌的表现，接下来是结果揭晓阶段。

X @swyx

Hermes Agent：完成的工作流自动变成可复用 Skill

AINews 详细报道了 Hermes Agent 的核心设计：它不只是工具调用，而是持续的 skill 积累。skill 在这里指"可复用的工作流模块"：一个任务完成后，系统会判断这个工作流是否值得复用，如果是，自动封装成 Skill 供下次调用。

有人展示它自主补填历史数据、更新 cron 任务（定时任务），然后把整个流程存成技能。这和"每次对话从零开始"的 ChatGPT 范式是完全不同的设计哲学。

Agent 能不能学习，不是靠微调，而是靠有没有工作流记忆层。

人文与哲学

窑火与量子：关系是现实的基本单位

Maria Popova 把卡罗·罗韦利的量子力学（Helgoland）和陶瓷釉药放在一起讲：两者都表明，物体的属性不是孤立存在的，只在关系中才能确定。量子粒子的"颜色"取决于它与什么相互作用，陶瓷的釉色取决于它经历了什么样的窑火。

关系不是装饰，是本质。

这个框架放在 AI 时代也挺有趣：我们讨论模型能力时，能力本身可能是一个关系属性，而不是模型本身的固有属性。换句话说，同一个模型在不同工具、不同上下文里表现出的能力，可能本质上是两个东西。

AI 进展

GameWorld：多模态游戏 Agent 的标准化评估框架

现状：Agent benchmark（测试基准）现在是一团乱麻。每个团队用自己的任务集、自己的评估逻辑，结果无法横向比较，谁好谁差全靠自说自话。

问题：如何建立一个标准化、可验证、可复现的多模态 Agent 评估框架？

解决方案：新加坡国立大学提出 GameWorld，把游戏环境做成标准化测试床：统一 API、可控难度、确定性验证。任何 Agent 都可以在同一条件下被测试，结果可复现。

价值：游戏是理想的测试环境。规则清晰、目标可量化、难度可以精确调节，比"让 Agent 完成文书任务"的 benchmark 更难造假。

未来方向：如果 GameWorld 被广泛采用，Agent 的能力进步将首次可以被系统跟踪，而不是靠论文里的精选样本。今日 HuggingFace 热度榜首（103 upvotes）。

Seedance 2.0：ByteDance 专攻视频生成的世界一致性

现状：AI 视频生成已经过了"能不能生成"这关，当下瓶颈是生成出来的世界是否自洽。多个角色在场景里运动时物理行为要合理，跨镜头的同一物体外观要一致，长片段不能出现无中生有或凭空消失。

问题：怎么让视频生成模型在复杂场景中保持一致性？

解决方案：ByteDance Seed 团队的 Seedance 2.0 针对这个"世界复杂性"问题做了专项训练，聚焦复杂物理交互、多角色协调和长时序一致性。

价值：直接的。生成内容从"需要大量人工修复"推向"可直接使用"。

未来方向：视觉一致性这个方向一旦过关，视频生成才真正进入创意工具链。（96 upvotes）

HuggingFace · arXiv 2604.14148

RationalRewards：把推理 scaling 的逻辑延伸到视觉生成

现状：语言模型的 test-time compute scaling（推理阶段多算一会儿、投入更多算力得到更好答案）已经被证明有效。

问题：这个逻辑在视觉生成模型上几乎没有。模型训练时学到的推理能力，在生成图像/视频时被完全浪费了。

解决方案：RationalRewards 用推理奖励信号打通这条路，让模型在推理阶段也能用 reasoning 持续改善输出质量，而不只是训练时受益。

价值：如果可行，意味着视觉生成质量也可以通过投入更多算力在推理时提升，不必重新训练模型。这是一个架构方向，不只是一篇论文。

未来方向：值得保留怀疑。实际计算开销和提升幅度的权衡还需要独立验证。（90 upvotes）
