Claude Opus 4.7 发布：SWE-bench Verified 87.6%，视觉翻3倍，xhigh 档位默认开

Anthropic 今天发布 Claude Opus 4.7，基准（benchmark，行业用来横向比较模型能力的标准测试集）成绩全面高于 4.6：SWE-bench Pro 64.3%（+11 分），SWE-bench Verified 87.6%（+7 分），ARC-AGI-2 75.83%，在 GDPval-AA 评测上拿到 1753 Elo，当前排名第一。Cursor 内部评测从 58% 跳到 70%，Notion 内部 eval 提升 14%。

三个技术变化值得记住。第一，新 tokenizer（分词器，决定模型如何把文本切成可处理单元）：同一段文字会比 4.6 多消耗最多 35% 的 token，但 Anthropic 同步上调了所有订阅套餐的用量上限。定价仍是 $5/$25 每百万 token 输入/输出。第二，视觉分辨率：支持最长边 2576 像素（约 3.75 百万像素），比之前的 Claude 模型提升约 3 倍，不再压缩高分辨率截图，computer use（用 AI 直接操控电脑界面的能力）场景直接受益。第三，新推理档位 xhigh：介于 high 和 max 之间，Claude Code 默认使用此档位。

OpenAI 发布 GPT-Rosalind：生物学专用 LLM，连接公开数据库，偏向"否定坏靶点"

GPT-Rosalind 是 OpenAI 专门为生命科学研发的模型，覆盖生物学、药物发现和转化医学（把基础研究转化成临床治疗的过程）。它能直连主要公开生物数据库，协助研究者做文献综述、假说探索到实验设计的全链路推进，并被特别调整为"更倾向于否定坏的药物靶点"，减少误导研究者浪费时间在死胡同上。

访问目前限定美国境内特定机构，通过 OpenAI 的受信访问部署结构（Trusted Access Deployment）管控。发布时间与 Opus 4.7 同天，但性质不同：Rosalind 不是通用模型的迭代，而是垂直场景的专用系统，意在压缩药物早期发现动辄 10-15 年的时间线。

Tencent HY-World 2.0 开源：文字/图像/视频一键生成完整可编辑 3D 世界

Tencent 发布并开源 HunyuanWorld 2.0，接受文本、图像、视频作为输入，输出完整 3D 环境，格式包括 mesh（三角网格）、点云和 Gaussian splat（高斯点云渲染格式），可直接导入 Unity 和 Unreal 等主流引擎。整个流程分多阶段：先生成全景场景，再规划导航路径，再扩展环境，最后拼装成连贯世界，用于游戏关卡原型、地图生成和资产制作。

与之前大多数 3D 生成模型只能输出几秒短片或静态场景不同，HY-World 2.0 的输出是可编辑的生产级资产。开源意味着个人开发者和独立工作室也能直接使用。

Cloudflare Agents Week：14 家供应商统一推理层 + Email Service 公测 + Artifacts 存储

Cloudflare 本周密集发布 Agent 基础设施更新。三个核心：第一，统一推理层（Unified Inference Layer），开发者一次接入就能调用 14 家模型供应商，不用为每家写一套调用逻辑。第二，Email Service 进入公测，Agent 可以原生收发和处理邮件，不依赖第三方 API。第三，Artifacts：Git 兼容的版本化存储，专为 Agent 状态和产物管理设计。

这三个更新合在一起，指向同一个方向：Cloudflare 在为 Agent 的生产化运行搭水管——不是卖模型，而是卖"让 Agent 跑起来所需的基础设施层"。

AI 方向观点

Cat Wu（Anthropic 工程师）：Opus 4.7 要当成"可委托的工程师"用，不是结对编程伙伴

Cat Wu 在 Opus 4.7 发布当天发了一条使用指引，核心逻辑是：这个模型设计上倾向于自主完成任务，给它完整的目标、约束和验收标准，然后走开。不要每一步都干预（pair programming，即结对编程，两个人/人和 AI 实时共同写代码），因为这会打断模型的自主执行节奏。

她还建议把验证逻辑写进 claude.md（Claude Code 项目的配置文件）或 skill（可复用的指令模板）里，让模型知道"怎么证明自己做完了"，而不是靠人在旁边实时确认。Claude Code 默认把 Opus 4.7 设成 xhigh（最高推理档位之一）也印证了这个定位：高推理成本换自主完成率。

Anthropic 把 Opus 4.7 推向了"异步委托"模式，这要求用户对 AI 的信任阈值和任务拆解方式都做出改变。

Boris Cherny（Anthropic）：MRCR 不再是有效的长上下文基准，Graphwalks 是替代

Opus 4.7 在 MRCR（Multi-Round Context Recall，通过在长上下文中隐藏干扰信息来测试模型记忆力的评测方式）上的得分低于 4.6，引发外界质疑。Anthropic 工程师 Boris Cherny 直接回应：MRCR 会被专门为"藏针游戏"设计的技巧过度优化，衡量的不是真实的推理能力。他给出了 Graphwalks（一种通过图结构推理任务来测试长上下文应用理解力的新指标）的数据：Opus 4.6 是 38.7%，Opus 4.7 是 58.6%，涨幅明显。

这件事本身揭示了一个评测行业的结构性问题：当某个 benchmark 被普遍采用后，模型厂商很容易专门针对它调优，导致数字好看但实际能力没涨。Anthropic 选择在系统卡里保留 MRCR 数据（"为了科学诚实"），但同时主动推动迁移到新指标，是一个不常见的公开自我挑战。

下一代 benchmark 大战的赌注不是谁的数字高，而是谁定义了什么叫"真实能力"。

产品与商业

Peter Yang（Follow 联合创始人）：构建 AI skill 时，同时跑一个 eval agent 做 yes/no 检查

Peter Yang 发推说他现在构建所有 AI skill（可复用的指令模板或工具）时，都会让 AI 同时启动一个独立的 eval agent（评估用的子 Agent，专门对主 Agent 的每个输出做是非判断）。这个 eval agent 的工作非常窄：只回答"这个输出通过了吗"，不做其他事。

这个模式的成本很低，但信号价值很高：你不需要事后人工审核每一条输出，eval agent 帮你把质量检查内嵌在执行流程里。这也让 skill 本身更容易迭代，因为每次修改后立刻有自动化的合格/不合格反馈。对做 Agent workflow 的 PM 来说，这是一个可以今天就落地的设计模式。

从确定性到概率性：AI 代码时代，生成变便宜了，但验证没有

Tim Davis 的这篇文章提出一个当前软件开发的核心矛盾：确定性工程（你写代码、测试它、理解它做什么）正在被概率性工程（大段代码由随机系统生成、在时间压力下审查、集成进没有人完整设计过的系统）取代。生成成本已经接近零，但验证成本没有降。

当 AI 产生的代码量超过人类注意力可以有效检查的上限时，bug 就会漏过去，因为产出速度已经超过了人工检验的上限，而负责审查的 AI 也会漏掉很多。他把 AI 比作"24-7 员工"：永不疲倦，但也永不犯错地存在——这个假设本身就是危险的。

这对工程师、PM 和产品团队的意味是：你们面临的新问题不是"怎么写代码"，而是"怎么验证你不完全理解的代码"。投资 review 流程、测试基础设施和可观测性，边际价值正在快速上升。

AI 把代码生成变成了水龙头，但没有人给验证装个水表。

片段经济：TBPN 每集 7000 播，每条剪辑 25.7 万播，片段已是主要产品

AI 进展

Physical Intelligence π0.7：机器人大脑首次可语言引导完成未经训练的任务

现状：现有机器人模型绑定训练任务集，机器人只能做它"见过"的事。给新任务就得收集数据、重新训练，周期长、成本高。

问题：真实场景任务千变万化，固定训练集的机器人在部署后很快就遇到覆盖盲区。

解决方案：π0.7 通过语言引导（verbal coaching，用自然语言口头指导机器人，而不是重新训练模型）实现零样本任务泛化（zero-shot generalization，意为第一次遇到某个任务、没有对应训练数据，也能尝试完成）。模型不需要额外数据采集或重新训练，接到口头描述的新任务后直接尝试执行。

价值：这是机器人从"会背题"到"会听话"方向上的一个早期但有意义的验证。从 demo 视频来看，复杂多步任务从单条高层指令自主完成还做不到，但单步新任务已经有效。

未来方向：从"语言引导单步"到"语言引导多步自主流程"，是下一个明确的里程碑。目前仍处于研究阶段，尚未产品化。

Opus 4.7 技术拆解：新 tokenizer + 3.75MP 视觉 + xhigh 档位，三层叠加的实质

现状：从 4.6 到 4.7，表面上是版本迭代，内部有三处结构变化同时发生，各自独立但效果叠加。

新 tokenizer：分词器换了，意味着这可能是从新基础训练起步的模型（而非仅微调），token 消耗量最多增加 35%，但推理效率提升抵消了部分成本，Anthropic 同步提高了所有套餐用量上限。LlamaIndex 的独立测试显示图表理解从 13.5% 跳到 55.8%，但布局理解略有下降，说明提升是非均匀的。

视觉升级：支持图像最长边 2576 像素（约 3.75MP），之前约 1.2MP。高分辨率截图不再被压缩，computer use（AI 直接操控电脑图形界面）和复杂图表分析直接受益。Anthropic 工程师 Alex Albert 特别指出 UI/slides/docs 的输出"品味"也有提升。

xhigh 档位：新增推理努力级别 xhigh，介于 high 和 max 之间。Claude Code 把 Opus 4.7 默认设为此档位，意味着 Anthropic 在用更多推理计算（test-time compute，推理时消耗的算力，区别于训练时算力）换取任务完成质量，而非靠参数量取胜。

未来方向：tokenizer 换代通常是基础模型的代际信号，外界猜测 4.7 可能是 Mythos（Anthropic 目前仅对安全研究伙伴开放的更强模型）的蒸馏版本，Anthropic 未正式确认。
