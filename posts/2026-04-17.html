Anthropic 营收首超 OpenAI：$30B ARR，企业客户双月破千

Anthropic 宣布年化营收（ARR，即把当前月收入换算成全年数字的行业通用计量方式）达到 $300 亿，首次超过 OpenAI 的 $250 亿。增长曲线异常陡峭：从 2025 年 1 月的 $10 亿到现在，只用了 15 个月。今年 2 月 Series G 融资时，每年付超过百万美元的企业客户有 500 家，今天超过了 1000 家，不到两个月翻了一倍。

市场份额方面，Anthropic 现在占企业 LLM API 支出的 40%，OpenAI 从 2023 年的 50% 降到了 27%。这不是产品能力的倒转，是商业模式的分歧：Anthropic 80% 营收来自企业客户，OpenAI 消费者比重更高。企业客户粘性强、可预期，消费者容易流动。

Meta 双线押注：$2B 收购 Manus + 接收 Dreamer 创始团队

Meta 用约 $20 亿收购了 Manus（一个能自主完成从查资料到操作网页等全流程任务的 AI Agent 产品），同时全盘接收了 Dreamer AI 的创始团队，后者原本在做面向普通用户的个人 Agent OS（操作系统层）。Manus 当时估值约 $5 亿，本轮溢价约 4 倍，ARR 约 $1 亿。

Dreamer 此前拿了 $5600 万融资，核心命题是"非技术用户也能用的个人 AI Agent"，整个创始团队已整建制并入 Meta Superintelligence Labs。两笔交易合在一起，意图清晰：一个负责任务执行（Manus），一个负责界面与用户端体验（Dreamer），目标是拼出 Meta 自己的 AI OS 层。

Windsurf 2.0 发布：Spaces 组织单位 + Kanban 管理面板 + SWE-grep

Cognition AI 发布 Windsurf 2.0，三个核心更新。第一，Spaces：把一个项目的所有 Agent 会话、PR、文件和上下文打包成一个单位，新建 Agent 时自动继承项目背景，不用每次重头交代。第二，Agent Command Center：看板（Kanban，一种用列表可视化追踪任务进度的界面）界面，可以同时看到所有 Agent 是跑着、卡住了还是在等待审核。第三，集成 Devin：把 Devin（Cognition 的自主编程 Agent）嵌入 IDE，一键把任务转给云端 Devin 执行，关掉电脑也不影响进度。

同时发布了 SWE-grep 和 SWE-grep-mini，两个专门做代码上下文检索的子模型，速度比 frontier 模型快约 20 倍，每轮最多并发 8 个工具调用（详见 AI 进展）。

AI 方向观点

Swyx（Latent Space 联合创始人）：Meta 的赌注进入"turn"牌阶段

Swyx 发推说，Meta 的 AI 叙事走到了一个关键节点："看到了 flop，现在正在看 turn"。这是德州扑克比喻：flop 是前三张公共牌，turn 是第四张，意思是前期的下注和初步结果（Llama 4、Manus 收购、Zuck 亲自编码）都已经亮出来了，现在是要看下一张验收牌。

他具体列出的信号：Meta 的 hiring momentum（招聘节奏）、Zuck 本人重新开始写代码、新一代模型能力提升、以及连续两笔 Agent 收购，认为这一轮 Meta 的 AI 押注比外界判断的要认真得多。

Llama 4 是 Meta 的 flop，接下来的 turn 是 Agent 能力，river 是 AI OS 落地——牌桌上的局还没结束。

Aaron Levie（Box CEO）：AI 不消灭工作，它把瓶颈换一个地方

Levie 引用了一篇 FT 报道说，AI 为什么在很多行业会创造新工作：你用 AI 加速了一个环节，就会在流程的另一个地方撞上新的瓶颈，而这个新瓶颈还需要人。他举了法律服务的例子：AI 批量生成法律信函和专利草稿，反而让审核这些文件的律师更忙了，因为需求量上来了。

类似模式在软件工程（更多代码 = 更多安全审查）和医疗（更快诊断 = 更多后续处理）都在发生。这背后是经济学上的 Jevons 悖论（Jevons Paradox）：当某种资源变得更高效，总消耗量反而增加，因为需求也跟着扩大。

AI 是需求放大器，不是需求终结者，但它会把你的护城河从"会做"变成"做得快还做得对"。

产品与商业

Anthropic 赢 OpenAI 的商业逻辑：训练成本 1/4，Claude Code 独撑 $25亿

同样做 frontier 模型，Anthropic 的训练花费约是 OpenAI 的四分之一。这不一定意味着他们的模型更便宜训练，更可能是产品路线的差异：Anthropic 把精力集中在企业 API 和工具链，而不是 ChatGPT 这样的消费级前端。

结果就是：Claude Code 一个产品，年化营收已经到 $25 亿，成为 Anthropic 增长最快的收入来源。对比整体 $300 亿 ARR，一个 CLI 工具（命令行界面工具，就是在终端里敲命令用的）贡献了约 8% 的营收，这个比例非常高。背后是工程师作为付费用户的消费力，以及 Agent 化工具从"方便"变成"必须用"的临界点越过了。

Anthropic 的赌注不是做最好的聊天机器人，而是做工程师离不开的基础设施。

SaaStr

Claude Code auto-compact：1M 上下文是双刃剑，在 60% 时主动出手

Claude Code 新版自动压缩功能（auto-compact：当对话历史快撑满上下文窗口时，自动把旧对话压缩成摘要，让 Agent 继续工作而不用重开）现在可以自定义触发阈值。Thariq（@trq212）的建议：1M 上下文窗口不是越大越好，在 60% 左右就手动跑一次 /compact 命令，主动压缩，而不是等自动触发。

原因是上下文越长，模型越容易被早期无关信息带偏；压缩本身也有信息损耗，在对话密度最高的时间点压缩比积累了很多废话再压缩损失更少。这跟"早 review 比晚 review 省力"是同一个逻辑。

Garry Tan（Y Combinator CEO）：GBrain 多 Agent 协作经实测稳定，关键是 git + postgres

Garry Tan 发推说 GBrain（他自己在用的多 Agent 工作系统）配合 git 和 postgres 同时运行多个 Agent 协作"效果非常好"。git（版本控制工具，记录代码每一次改动）管代码变更，postgres（一种关系型数据库）管任务状态和上下文共享，两者配合让多个 Agent 不互相踩踏。

人文与哲学

Peter Yang：从中国 AI 公司归来的 15 个观察

Peter Yang 走访了北京、上海几家头部 AI 公司，写了 15 条观察，有几条值得停下来想一想。工作节奏：11 点到公司，干到 11 点走，中间外卖+奶茶送到工位，主要原因是维持这种节奏的很多是还没孩子的年轻人。年轻文化：这一代不太喝酒、不怎么出去玩，不是因为纪律，是生活本来就长这样。

AI 工具：这些公司的工程师全用顶级美国 AI 工具，Claude Code 是提到最多的，走 VPN（翻墙工具）。政府角色：地方政府之间互相竞争，都想把最好的 AI 创始人吸引过来，给地、给钱、给政策。生活成本：住家保姆月薪约 $1500，私人大厨时薪约 $7，滴滴大多数订单 $3-5。

这种不对称的生活成本背后是一个现实：中国 AI 创业者能把几乎所有精力都放在工作上，因为生活的其他摩擦被廉价服务抹平了。这在结构上比较难复制。

Zara Zhang（Follow 联合创始人）：和 AI 模型"玩"就是认真的工作

Zara Zhang 发推说：很多人把"玩"AI 模型当成不正经的事，但她的体验是，那些随手试试的时间往往是一天里信息密度最高的时间段。实验本身就是最快的学习方式，不需要一个正式任务作为理由才能开始探索。

AI 进展

SWE-grep / SWE-grep-mini：强化学习训练的快速代码检索子模型

现状：多 Agent 并行工作时，最大的速度瓶颈之一是"找上下文"——让模型搞清楚当前代码库里的结构、接口、依赖。这步慢，整个流水线就慢。

问题：frontier 模型（指当前最强的大模型）虽然检索能力强，但太重，用来做这种高频、短任务非常浪费。需要一个又快又准的专用子模型。

解决方案：Cognition 用强化学习（Reinforcement Learning，通过奖惩反馈让模型自己学会策略，不需要人工标注每一步怎么做）专门训练了 SWE-grep 和轻量版 SWE-grep-mini，专攻多轮上下文检索。每轮最多同时发出 8 个工具调用，最多跑 4 轮，找完停手。

价值：在 Windsurf 的实测中，Fast Context 子 Agent（子模型负责的辅助模块）速度比用 frontier 模型快 20 倍，同时检索质量保持对齐。意味着并行工程师模式（一个人同时驱动几十个 Agent）从理论变成了可用的日常工具。

未来方向：这个思路可以推广：训练专用的轻量子模型接管流水线中的特定高频任务，而不是用一个全能大模型做所有事。SWE-grep 是这个方向的第一个有说服力的案例。

Cognition Blog · Hacker News

DeepAgent（WWW '26）：通用推理 Agent + 可扩展工具集

现状：大多数 Agent 系统工具集是固定的：开发者预先定义好 Agent 可以调用哪些工具，Agent 只能在这个范围内行动。

问题：真实任务需要的工具组合千变万化，固定工具集要么太少（Agent 做不了）、要么太多（推理效率低下）。

解决方案：DeepAgent 用 ToolPO（一种强化学习训练方法，用工具模拟器生成大规模真实 API 工具调用的训练数据，让模型学会按需发现和使用工具）做端到端训练。模型在推理时能主动发现新工具、执行、然后把这次工具使用的经验打包存进记忆，下次类似任务直接调用，不需要重新探索。

价值：工具记忆层让 Agent 越用越聪明，不是靠重新训练，而是靠经验积累。这和人类的技能增长模式更接近。在 WWW '26（2026年国际万维网大会，三月底至四月中旬在迪拜举行）上发表，同行评审已完成。

未来方向：ToolPO 方法能否迁移到其他 Agent 训练场景，以及工具记忆的长期一致性问题，是接下来的验证重点。
